# Общая информация

| Параметр          | Значение                                                         |
| ----------------- | ---------------------------------------------------------------- |
| **Система**       | «Струна-5» — охранно-пожарная сигнализация (НПФ «Интеграл+», РФ) |
| **СУБД**          | Firebird 2.1                                                     |
| **Период данных** | 2017–2019 (архив)                                                |
| **Объём**         | ~650 тыс. событий, ~67 тыс. инцидентов, ~220 объектов            |
| **Доступ**        | Локальный файл `.fdb` или через `gbak` restore                   |

## Таблицы и поля (только то, что берем в работу)

### `DEFOBJ` — Объекты (справочник)

| Поле          | Тип          | Описание               | Примечание                           |
| ------------- | ------------ | ---------------------- | ------------------------------------ |
| `GRP`         | SMALLINT     | Номер группы           | Часть составного ключа               |
| `MDM`         | SMALLINT     | Номер ячейки           | Часть составного ключа               |
| `NAIMOBJ`     | VARCHAR(50)  | Название объекта       | Для отображения                      |
| `ADROBJ`      | VARCHAR(100) | Адрес объекта          | Для гео-аналитики                    |
| `KODORG`      | INTEGER      | Код организации        | FK → CLIENTS (не загружаем)          |
| `KATEG`       | SMALLINT     | Код категории          | 0=объект, 1=квартира, 2=МХИГ         |
| `TIPOBJ`      | VARCHAR(20)  | Детальный тип          | «Объект», «Доп. оборудование» и т.д. |
| `GUARD`       | SMALLINT     | Код статуса охраны     | 0=исключён, 1=охрана, 2=прогон       |
| `BLKHARDWARE` | VARCHAR(20)  | Код оборудования       | FK → SP_BLKHARD                      |
| `CREATED`     | DATE         | Дата создания карточки | Для анализа возраста                 |
| `DATAZAKL`    | DATE         | Дата договора          | Для анализа клиентов                 |
| `NDOG`        | VARCHAR(20)  | Номер договора         | Юридическая привязка                 |

**Связи:**

DEFOBJ.KATEG → PROPERTY.KOD (название категории)
DEFOBJ.GUARD → SP_STATUS.KOD (название статуса)
DEFOBJ.BLKHARDWARE → SP_BLKHARD.BLKHARDWARE (название оборудования)

**Особенности:**

- Составной ключ: `GRP + MDM` → уникальный объект
- Много полей с низкой заполненностью — исключены из миграции

### `DATA` — События (факты)

| Поле         | Тип       | Описание                 | Примечание                  |
| ------------ | --------- | ------------------------ | --------------------------- |
| `EVENTID`    | INTEGER   | Уникальный ID события    | Primary Key, для инкремента |
| `DTTM`       | TIMESTAMP | Время события на объекте | Основная временная метка    |
| `GRP`, `MDM` | SMALLINT  | Ключ объекта             | → `DEFOBJ`                  |
| `MESSLOW`    | SMALLINT  | Код типа события         | → `MESS`                    |

**Объём:** ~650 тыс. записей  
**Период:** 2017–2019

**Особенности:**

- Нет прямой связи с `ALARM` — только через `ALARM_EVENT`
- **~5% событий без объекта в DEFOBJ** (см. Data Quality)

### `MESS` — Типы событий (справочник)

| Поле        | Тип         | Описание         | Примечание                  |
| ----------- | ----------- | ---------------- | --------------------------- |
| `MESSLOW`   | SMALLINT    | Код события      | Primary Key                 |
| `ALARMMESS` | VARCHAR(50) | Описание события | «Тревога», «Разряд батареи» |
| `CATEGORY`  | SMALLINT    | Код категории    | → `MESS_CAT.ID`             |
| `ISALARM`   | SMALLINT    | Флаг тревоги     | `'*'` — TRUE, `' '` — FALSE |

**Объём:** 238 записей

**Особенности:**

- `ISALARM = '*'` — legacy-формат, заменять на `1` в ETL

### `MESS_CAT` — Категории событий

| Поле   | Тип         | Описание                            |
| ------ | ----------- | ----------------------------------- |
| `ID`   | SMALLINT    | Код категории                       |
| `NAME` | VARCHAR(30) | Название: «Связь», «Питание», и т.д |

**Объём:** 8 записей

### `ALARM` — Инциденты (факты)

| Поле         | Тип       | Описание                | Примечание                     |
| ------------ | --------- | ----------------------- | ------------------------------ |
| `ALARMID`    | INTEGER   | Уникальный ID инцидента | Primary Key                    |
| `GRP`, `MDM` | SMALLINT  | Ключ объекта            | → `DEFOBJ`                     |
| `CREATED`    | TIMESTAMP | Создание инцидента      | 100% заполнено                 |
| `PROCESSED`  | TIMESTAMP | Закрытие инцидента      | 100% заполнено, основная метка |
| `SENT`       | TIMESTAMP | Отправка наряда         | ~3.5% заполнено (только выезд) |
| `ARRIVED`    | TIMESTAMP | Прибытие группы         | ~2.4% заполнено                |
| `RESULT`     | SMALLINT  | Исход инцидента         | → `OTVET.KOD`, 99.8% заполнено |

**Объём:** ~67 000 записей

**Особенности:**

- 96% инцидентов закрываются без выезда (`SENT/ARRIVED` пустые)
- `PROCESSED` — надёжная метка завершения (всегда есть)
- `RESULT`: коды `-3..27`, некоторые без описания в `OTVET`

### `ALARM_EVENT` — Связка событий и инцидентов

| Поле      | Тип     | Описание          |
| --------- | ------- | ----------------- |
| `EVENTID` | INTEGER | → `DATA.EVENTID`  |
| `ALARMID` | INTEGER | → `ALARM.ALARMID` |

**Назначение:** Точная связь без эвристик по времени

## Data Quality:

| Проблема                | Где                               | Проявление                                 | Решение в ETL                     |
| ----------------------- | --------------------------------- | ------------------------------------------ | --------------------------------- |
| `ISALARM = '*'`         | `MESS`                            | Строка вместо числа                        | `replace('*', '1')` → `int`       |
| Составной ключ          | `DEFOBJ`, `DATA`, `ALARM`         | `GRP+MDM` вместо одного ID                 | Хеширование → `object_id_anon`    |
| Кодировка               | Все таблицы                       | Кракозябры при чтении                      | `charset='WIN1251'` в подключении |
| ПДн                     | `DEFOBJ.NAIMOBJ`, `DEFOBJ.ADROBJ` | Наименования объектов и их реальные адреса | Маскировка для публикации         |
| **События без объекта** | `DATA.GRP+MDM`                    | **~39 592 события (5%) без DEFOBJ**        | **Флаг `has_object_ref`**         |
| **Тестовые ключи**      | `DATA.GRP=0, MDM=0/1`             | **~16 000 событий (системные)**            | **Маркер `test_default`**         |

  • fact_alarm_events (ALARM_EVENT)

  • dim_objects (DEFOBJ + PROPERTY + SP_STATUS + SP_BLKHARD)
  • dim_event_types (MESS + MESS_CAT)
  • dim_outcomes (OTVET)
